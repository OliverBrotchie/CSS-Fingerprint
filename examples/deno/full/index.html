Captured fonts cannot be displayed to a user without a backend. (WIP)

Every font not installed on device will send a request and by comparing differences between the requests and the full list of fonts, we can draw conclusions about what fonts are installed.

Fonts tested can be found here.

How this works

This page detects specific features about a browser using CSS; exfiltrating this information by exploiting URL() tags. Open dev tools. You will note the browser is making requests to URLs that correspond to infomation specific to your device.

Since this is a simple demo, it is not hooked up to an API and hence these requests are returning errors.(WIP)


Technique inspired by:
Takei, N., Saito, T., Takasu, K. and Yamada, T., 2015, November. Web browser fingerprinting using only cascading style sheets. In 2015 10th International Conference on Broadband and Wireless Computing, Communication and Applications (BWCCA) (pp. 57-63). IEEE.
