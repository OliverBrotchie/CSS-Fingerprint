Captured fonts cannot be displayed to a user without a backend. (WIP)

Every font that isnt installed on the device will send out a request and by finding the difference between the requests and the full list of fonts, we can determine what is.

Fonts tested can be found here.

How this works

This works by detecting spesific features of the browser using CSS; exfiltrating this infomation by exploiting url() tags. As you can see if you open dev tools, the browser is making requests to urls that correspond to spesific device infomation.

This is a simple demo and is not hooked up to an api and hense these requests are returning errors.(WIP)


Technique inspired by:
Takei, N., Saito, T., Takasu, K. and Yamada, T., 2015, November. Web browser fingerprinting using only cascading style sheets. In 2015 10th International Conference on Broadband and Wireless Computing, Communication and Applications (BWCCA) (pp. 57-63). IEEE.
